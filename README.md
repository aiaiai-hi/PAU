# Дашборд мониторинга качества руководителей ВСП (концепция D)

Streamlit-приложение по согласованному прототипу: три уровня (Банк / Филиал / Доп. офис),
под-вкладки «Обзор / Детализация по отклонениям / Ассистент», фирменная зелёная гамма,
визуализации на Plotly (спидометр, динамика по зонам, карта-хитмап Н/М/К, бары) вместо «голых» таблиц.

## Запуск

```bash
pip install -r requirements.txt
streamlit run app.py
```

Откроется в браузере (по умолчанию http://localhost:8501).

## Структура

| Файл | Назначение |
|------|-----------|
| `app.py` | Интерфейс: сайдбар-фильтры, переключатель уровней, под-вкладки, навигация (проваливание), стили. |
| `viz.py` | Все Plotly-визуализации (спидометр, динамика, хитмап, бары, сравнение, пончик). |
| `mock_data.py` | Мок-данные по структуре исходного файла + агрегаты для всех уровней и ассистента. |
| `requirements.txt` | Зависимости. |

## Что где

- **Сайдбар** — «Период статистики» (Неделя / Месяц / Квартал + Год) для вкладки «Обзор», блок «Организация» (Рег.директор, Филиал, Рег.менеджер), поиск, доп. фильтры.
- **Банк** — KPI (средняя интегральная оценка, офисы по зонам), рейтинг филиалов (стек-бар по зонам) с кнопками проваливания, топ отклонений, пончик распределения сети.
- **Филиал** — «Обзор» (KPI, ТОП-5 проседающих, сравнение с Банком, бары офисов с кнопками), «Детализация» (календарь + карта Н/М/К, регулярные / стабильные, ТОП-5 по периодам), «Ассистент».
- **Доп. офис** — «Обзор» (зелёная плашка «Место в рейтинге» + спидометр «Интегральная оценка» рядом, динамика по зонам, ТОП-3 с баннером, сравнение, множители), «Детализация», «Ассистент».
- **Календарь** в «Детализации» задаёт период; карта показывает все показатели совокупно: недельные (Н№), месячные (М№), квартальные (К№) — в хронологическом порядке (недели месяца → месяц → квартал).

## Подключение реальных данных

Сейчас данные генерируются в `mock_data.py`. Чтобы подключить ваш файл:

1. В `mock_data.py` замените тело `load_dataframe()` на чтение вашего источника, например:
   ```python
   df = pd.read_excel("data.xlsx")   # или read_csv(...)
   ```
2. Приведите имена столбцов к тем, что использует приложение (Рег.директор, Региональный филиал,
   `aggregation_level`, Наименование ВСП, Наименование отклонения, `report_week_number` / month / quarter,
   Вес отклонения в рейтинге, Пораженность отклонением %, Средний счётчик (срок жизни),
   Оценка работы по отклонению, Интегральная оценка качества работы, Позиция в рейтинге, Светофор).
3. Перепишите функции-агрегаторы (`branches_summary`, `bank_summary`, `office_top_deviations`,
   `detail_matrix` и т.д.) на `groupby` по вашему `df`. Сигнатуры менять не нужно — UI и графики останутся прежними.

## Ассистент

По умолчанию ассистент отвечает по эвристике на основе данных объекта (демо).
Чтобы он отвечал «по-настоящему» через Claude:

```bash
export ANTHROPIC_API_KEY=sk-ant-...
# по желанию: export CLAUDE_MODEL=claude-opus-4-8
streamlit run app.py
```

Если ключ задан, ассистент отправляет в Claude краткий контекст по объекту и вопрос пользователя
(см. `assistant_reply()` в `app.py`).
